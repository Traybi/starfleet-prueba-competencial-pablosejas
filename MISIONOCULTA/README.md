Ahora lo que teniamos que hacer era un find para buscar el siguiente paso, que en este caso es la mision oculta:

	"sudo find / -name mision_oculta.sh"

Para ver el contendido de la siguiente prueba hacemos un cat  y nos dice que hay que crear un script para poder tener informacion de algunas partes del sistema.  Todos los scripts creados están en el repositorio.

IMPORTANTE: a cada archivo .sh creado, tenemos que hacer un "sudo chmod +x "nombre_archivo"".

Una vez he creado todos los .sh por separado, he creado un menú para el capitán, el cuál cuando elije una opción, se ejcuta el script individual, por lo que nos queda un script muy simple.

¿Por qué es importante explorar el sistema más allá de lo que te dicen las instrucciones de clase?

Porque podemos encontrar información valiosa o atajos. Nos prepara para entornos reales en los cuales nosotros tengamos que indagar de verdad en un sistema. Podremos perfeccionar técnicas de búsqueda. Sobretodo, porque siguiendo unas instrucciones podemos caer en un bucle generado aposta, y si investigamos sobre el sistema, podemos ver que no acaba ahi y que era un bucle intencionado. Ir por otros caminos a veces nos puede hacer relantizarnos, pero muchas veces podemos encontrar archivos ocultos o encontrar archivos directamente saltando y evitando posibles trampas o entretenimientos.
